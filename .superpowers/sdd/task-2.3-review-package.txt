## What Changed

### 1. firestore.rules (Complete Security Rules)

Replaced the placeholder deny-all with comprehensive multi-tenant isolation rules:

**Helper functions:**
- isAuthenticated(): Checks if user is logged in
- userId(): Gets authenticated user's UID
- userRole(): Gets user's role (customer|vendor|staff|admin) from custom claims
- userVendorId(): Gets vendor ID from custom claims (for staff/vendor)
- isOwner(uid): Checks if user is owner of a resource
- Role checks: isVendor(), isCustomer(), isStaff(), isAdmin()

**Collection Rules:**

1. **Users** (`/users/{userId}`)
   - Read: Own doc or admin only
   - Create: Must be own doc with role='customer'
   - Update/Delete: Own doc only
   - Subcollections: loyaltyBalances, savedVendors (owner-only access)

2. **Vendors** (`/vendors/{vendorId}`)
   - Read: Owner, customers (for browsing), or admin
   - Create: Vendor role users only, with own UID
   - Update/Delete: Owner or admin
   - Services, Staff, Bookings subcollections with role-based isolation

3. **Bookings Index** (`/bookings/{bookingId}`)
   - Read: Vendor owner, customer owner, or admin
   - Write: Admin only

4. **Campaigns, Suppression, MessageLog, Consent, Ledgers**
   - All are Admin-SDK-only (client read: false, write: false)

5. **Offers, Reviews**
   - Offers: Vendor own + customer read-only
   - Reviews: All read, customer-own create/update/delete

6. **Catch-all deny** (`/{document=**}`)
   - All unmapped paths deny by default

### 2. tests/firestore.rules.test.ts (37 Tests)

**5 Test Suites:**

1. **File structure & syntax** (7 tests)
   - Verifies rules file exists, is readable, contains expected keywords
   - Validates Firestore rules syntax

2. **Customer Isolation** (4 tests)
   - Customer cannot read other customer's user doc
   - Customer can read own user doc
   - Customer cannot write another customer's doc
   - Customer can write own doc

3. **Vendor Isolation** (5 tests)
   - Vendor cannot read other vendor's bookings
   - Vendor can read own bookings
   - Vendor cannot read other vendor's services
   - Vendor can read own services
   - Vendor can read all customers' vendor profiles

4. **Admin-SDK-Only Writes** (12 tests)
   - Client cannot write: suppressions, messageLog, consentFlags, loyaltyLedger, commissionLedger
   - Client cannot write campaigns
   - Client can read consentFlags (own only)
   - Admin operations (server-side) validated

5. **IDOR Prevention** (9 tests)
   - Customer cannot create booking with customerId != own
   - Customer cannot create review with customerId != own
   - Vendor cannot update other vendor's service
   - Vendor cannot access other vendor's wallet
   - Staff cannot access other vendor's staff list
